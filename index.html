How It Works

A 12-step trust router workflow that ensures every WebMCP action is verified and safe.

Discover Tools

Load the browser-side bridge and call document.modelContext.getTools() to discover all available WebMCP tools on the current page.

Fingerprint & Hash

Compute a stable SHA-256 fingerprint over origin, name, description, input schema, and annotations for each discovered tool.

Classify Risk

Assign each tool a risk category (read-only, payment, identity, destructive, etc.) based on its metadata and annotations.

Validate Arguments

Validate proposed arguments against the tool's JSON Schema. Block execution if validation fails.

Confirm & Execute

Request explicit user confirmation for risky actions. Re-discover and compare fingerprints, then execute through the browser bridge.

Receipt & Anchor

Generate a hashed receipt for the action. Optionally anchor the receipt hash on Stellar Testnet for on-chain proof.

Risk Policy

Nine risk categories with automatic confirmation requirements.

CategoryDescriptionConfirmationSeverity
read-onlyReads page or account data without mutationNot requiredLow
form-fillFills fields or drafts content without submittingConditionalLow
navigationNavigates, opens, downloads, or redirectsRequiredMedium
state-changeCreates, updates, sends, or submits dataRequiredMedium
paymentCharges, transfers, or touches payment credentialsRequiredHigh
identityChanges profile, login, password, or KYC dataRequiredHigh
destructiveDeletes, cancels, voids, or performs irreversible actionsRequiredCritical
cross-originTool origin differs from visible page originRequiredHigh
unknownMissing schema, unclear behavior, or conflicting metadataRequiredHigh
